DESIGN CONCEPT
“Strategy in Motion”: The page is a sequence of 6 scroll scenes:
Scene 1 = Hero
Scenes 2–6 = One scene per sub-service
Each scene is full viewport (min-height: 100vh) and uses a centered “SceneCard” module. The SceneCard contains text + a minimal animated visual element (CSS or inline SVG), and both animate together.

SCROLL CHOREOGRAPHY (MUST IMPLEMENT WITH GSAP ScrollTrigger, scrubbed timelines)
For EVERY scene card implement a 3-phase scroll-linked timeline:
1) ENTER (first 25% of scene scroll):
   - Card rises from below with subtle perspective tilt (rotateX/rotateZ small), opacity 0→1
   - Headline reveals via mask/clip-path slide
   - Body/notes stagger in
2) FOCUS / TRANSFORM (middle 50%):
   - Card settles to neutral (tiny scale correction)
   - The scene’s “signal element” animates in a way that matches the sub-service theme (details below)
   - Subtle background accent shift (very minimal)
3) EXIT (last 25%):
   - Card drifts diagonally out + scales down slightly + fades
   - Next scene begins to appear underneath (deck feeling)

IMPORTANT: Do NOT pin the page. You may pin ONLY the SceneCard for a very short range (optional), but default to no pinning. The page must feel like continuous motion.
